# 2.0 Keperluan Umum Laman Web

Bagi mewujudkan dan mengekalkan platform maklumat digital UiTM yang profesional, mudah diakses dan konsisten, semua laman web di bawah ekosistem UiTM hendaklah mematuhi beberapa keperluan umum yang telah ditetapkan. Piawaian ini memastikan aspek kebolehgunaan, keselamatan, dan keseragaman di seluruh laman web UiTM termasuk laman web utama, fakulti, kampus cawangan, pusat tanggungjawab, akademi, dan pejabat.

## 2.1 Pengalaman Pengguna (UX) dan Akses

- Laman web hendaklah direka bentuk agar mudah digunakan, intuitif dan mempunyai navigasi yang jelas untuk semua pelawat.
- Setiap laman web mesti mematuhi piawaian aksesibiliti seperti _Web Content Accessibility Guidelines_ (WCAG) bagi memastikan akses yang saksama kepada pengguna berkeperluan khas.
- Reka bentuk responsif (_responsive design_) perlu dilaksanakan bagi memastikan paparan laman web serasi di semua peranti seperti komputer _desktop_, tablet, dan telefon pintar.

## 2.2 Penjenamaan dan Identiti Visual

- Logo rasmi, warna korporat, dan jenis huruf hendaklah digunakan secara konsisten di semua laman web.
- Susunan dan reka bentuk laman hendaklah sejajar dengan identiti korporat UiTM bagi mengekalkan pengiktirafan jenama institusi.
- Komponen antara muka (_UI components_) rasmi perlu diseragamkan bagi mewujudkan penampilan dan pengalaman pengguna yang seragam di semua halaman laman web UiTM.
- Templat laman web rasmi UiTM akan disediakan oleh Pejabat Komunikasi Strategik UiTM dan wajib dijadikan asas reka bentuk bagi semua laman web di bawah domain UiTM.

## 2.3 Pengurusan dan Pengemaskinian Kandungan

- Kandungan laman web hendaklah jelas, padat dan relevan, serta mematuhi piawaian komunikasi rasmi UiTM.
- Segala maklumat perlu disemak dan dikemas kini secara berkala agar mencerminkan perkembangan terkini seperti program akademik, pengumuman fakulti, hasil penyelidikan, perkhidmatan dan berita universiti.
- Setiap laman web hendaklah mempunyai proses semakan dan kelulusan kandungan (_content moderation and approval_) bagi memastikan kualiti dan ketepatan maklumat yang diterbitkan.

## 2.4 Keselamatan dan Perlindungan Data

- Pemasangan sebarang komponen atau pemalam (plugins) yang tidak disenaraikan dalam garis panduan ini **tidak dibenarkan**.
- Kebenaran folder (_folder permission_) mesti ditetapkan pada tahap yang betul; folder yang boleh ditulis secara global adalah dilarang (CHMOD:777).
- Sistem Pengurusan Kandungan (CMS) hendaklah dikemas kini setiap kali versi baharu dikeluarkan.
- Akaun Super Admin dan Administrator hendaklah dihadkan kepada individu tertentu sahaja. Sekiranya berlaku pertukaran webmaster, kata laluan mesti ditukar serta-merta oleh webmaster baharu.
- Gunakan kata laluan yang kukuh (minimum 12 aksara) dan ubah sekurang-kurangnya sekali setahun.
- Semua laman web mesti mematuhi garis panduan keselamatan digital yang dikeluarkan oleh Jabatan Digital UiTM.

## 2.5 Pengoptimuman Prestasi

- Setiap halaman mesti boleh dimuat turun dengan pantas dan dioptimumkan dengan mengurangkan penggunaan grafik, skrip dan data yang tidak perlu.
- Amalan Pengoptimuman Enjin Carian (SEO) perlu dilaksanakan bagi meningkatkan kebolehlihatan laman web di enjin carian utama.
- Laman web hendaklah diperiksa secara berkala untuk mengesan pepijat (_bugs_), pautan rosak (_broken links_) dan isu teknikal bagi memastikan kebolehpercayaan serta kestabilan sistem.

## 2.6 Integrasi dan Keserasian

- Setiap laman web UiTM mesti menyokong integrasi dengan laman rasmi UiTM dan pautan berkaitan lain, contohnya bagi laman fakulti, pautan ke portal pelajar, UFuture, dan platform berkaitan hendaklah disediakan.
- Laman web hendaklah serasi dengan pelayar utama seperti Google Chrome, Mozilla Firefox, Microsoft Edge dan Safari.
- Pautan media sosial dan platform luaran yang relevan boleh diintegrasikan secara lancar untuk memperkukuh komunikasi digital universiti.
